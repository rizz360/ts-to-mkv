# Security Policy

## Supported Versions

Security updates are provided for the latest release line.

| Version | Supported |
| ------- | --------- |
| latest  | Yes       |
| older   | No        |

## Reporting a Vulnerability

Please do not open public issues for security reports.

1. Open a private security advisory in GitHub (preferred).
2. Include impact, reproduction steps, and affected version/tag.
3. If possible, include logs or a minimal test case.

You will receive an acknowledgement within 5 business days.

## Disclosure Process

1. We acknowledge and triage the report.
2. We confirm impact and prepare a fix.
3. We publish a release with notes and mitigation guidance.
4. We credit the reporter if they want to be acknowledged.
